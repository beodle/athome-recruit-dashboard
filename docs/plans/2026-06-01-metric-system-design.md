# 채용 콘텐츠 대시보드 — 지표 체계 설계 (Metric System Design)

> 작성: 2026-06-01 · 브레인스토밍(superpowers) 결과 · 승인 완료
> 대상: `채용-인프라/대시보드/채용성과/dashboard-v2.html` (배포: beodle/athome-recruit-dashboard)

## Context (왜)
대시보드가 누더기처럼 느껴진 근본 원인은 "무엇을 측정할지"가 코어로 잡혀 있지 않아서였다. UI를 다시 그리기 전에 **지표 체계 자체를 단단히 정의**한다. 목적은 거창한 attribution·지원자 퍼널이 아니라, **운영자가 주간·월간으로 전체 성과를 한눈에 리뷰**할 수 있는 수준이다. 대시보드는 이 체계의 표현일 뿐이다.

## 목적 & 스코프
- **목적**: 전체 성과 리뷰(주간 + 월간). 비교 기준 = 직전 동일 기간.
- **스코프 인**: 인지 / 콘텐츠 성과 / 유입 품질.
- **스코프 아웃**: 지원 이후 단계(지원자 수·합격·입사·이탈), 콘텐츠→지원 인과 attribution, 지원 퍼널(공고상세→지원→완료) — **헤드라인·근거에서 제외**.
- **접근**: A(성과 스코어카드 + 근거 섹션) + B(지표 정의서).

## 지표 정의서 (B)
공통 규칙: 비교 = 직전 동일 기간(주간→직전 주, 월간→직전 월), 표기 ▲▼ %. LinkedIn은 시트·수동 의존 → 미갱신 구간은 **0이 아니라 공백**으로 표시.

**① 인지 (브랜딩이 자라는가)**
| 지표 | 정의/공식 | 출처 |
|---|---|---|
| LinkedIn 팔로워 | 기간 말 누적 / 순증=기간 신규 | LinkedIn(시트·수동) |
| LinkedIn 노출 | 기간 노출 합 | LinkedIn 시트/XLS |
| 채용페이지 방문 | GA4 활성 사용자 합 | GA4(자동) |
| 블로그 조회 | GA4 페이지뷰 합 | GA4(자동) |

**② 콘텐츠 성과 (무엇이 먹혔는가)**
| 지표 | 정의 | 출처 |
|---|---|---|
| 발행 수 | 기간 내 캘린더 발행 항목 수(채널 분해) | 콘텐츠 캘린더 시트 |
| 베스트 콘텐츠 | 게시물 랭킹 1위(노출/조회) + S·A 등급 수. **인덱스·카테고리 페이지 제외** | LinkedIn 분석 시트 / 블로그(GA4) |

**③ 유입 품질 (결이 맞는가)**
| 지표 | 정의/공식 | 출처 |
|---|---|---|
| 블로그 평균 참여 시간 | `userEngagementDuration ÷ 활성 사용자` (GA4 '평균 참여 시간'과 일치) | GA4(파이프라인 계산) |
| 신규 방문 비율 | `newUsers ÷ users` | GA4 |

## 화면 설계 (A)
**성과 스코어카드** — 페이지 최상단, 헤드라인 타일 6개(각 *값 · 직전 동일기간 ▲▼% · 미니 스파크라인*):
팔로워(순증) · LinkedIn 노출 · 채용페이지 방문 · 블로그 조회 · 발행 수(LI/블로그 분해) · 블로그 평균 참여 시간.
그 아래 **"이번 기간 핵심 변화" 한 줄** 자동 요약(변동 큰 1~2개).

**근거 섹션 3블록** (스코어카드 아래, 기존 v2 자산 재활용):
1. **인지 추세** — 노출(바)·블로그 조회(라인)·팔로워 누적(라인) 콤보 + ▸채널 주차 상세(아코디언)
2. **콘텐츠 성과** — 게시물 랭킹(등급·인덱스 제외) + 잘 먹힌 콘텐츠(S/A·훅, 시트 연동 시) + 발행↔영향도
3. **품질·유입** — 평균 참여 시간 추세 + 신규 방문 비율 + 유입 출처(도넛 · ▸전체 아코디언)

**제거**: 채용 퍼널 카드 전부.

## 디자인 원칙 (확정)
- 보더·패딩·배경 구분 최소화 — 위계는 **폰트 크기·굵기·여백**으로.
- 색은 **그레이스케일 우선**, 액센트(오렌지)는 실제 CTA(버튼)·팔로워 성장선·S/A 등급에만.
- 한국어 행간 1.6·자간 -0.006em, 큰 제목 자간 더 조임. **이탤릭 미사용.**

## 데이터·파이프라인 변경
- **`update.sh`**: 블로그 프로퍼티에서 `userEngagementDuration ÷ activeUsers`를 주차별로 계산 → `blog_avg_engagement` 필드로 굽기. (GA4 Data API엔 averageEngagementTime 단일 메트릭이 없어 계산 지표로 저장.)
- **프런트**: `blog_avg_engagement`를 그대로 표시(나눗셈 제거).
- **보존**: `__BAKED_DATA__`·`__LAST_UPDATED__` 마커, 내보내기(exportWithData), 범위 필터, python `*_to_dashboard.py` 호환.

## Verification
1. `update.sh` dry-run → `blog_avg_engagement` 주차별 생성 확인, 마커·BAKED 한 줄 보존.
2. `node --check`(스크립트 추출) 통과.
3. 브라우저: 스코어카드 6타일 + 핵심변화 한 줄 / 3섹션 / 주간·월간 토글 시 직전 동일기간 비교 / 아코디언.
4. 퍼널 카드 부재, 인덱스 페이지 랭킹 제외 확인.
5. 디자인 원칙(액센트 최소·이탤릭 0·여백 위계) 육안 검수.

## Out of scope (명시)
지원자 수·합격·입사·이탈·오퍼수락, 콘텐츠→지원 attribution, 공고/직무 세그먼트, 추가 채널(잡코리아·인스타·유튜브).
